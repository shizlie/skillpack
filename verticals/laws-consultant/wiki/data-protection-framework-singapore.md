# Data Protection Framework Singapore

**Summary**: Foundational PDPA concepts from current raw captures, focused on purpose, scope, and key definitional elements.  

**Sources**:
- Personal Data Protection Act 2012 - Singapore Statutes Online.md

**Last updated**: 2026-04-19

---

The PDPA purpose clause balances individual data protection rights with organisational need for reasonable collection/use/disclosure. (source: Personal Data Protection Act 2012 - Singapore Statutes Online.md)

The captured text defines personal data broadly as identifiable data, including via combined datasets likely accessible to an organisation. (source: Personal Data Protection Act 2012 - Singapore Statutes Online.md)

Application carve-outs captured include personal/domestic activity, employee-in-employment activity, public agencies, and certain data intermediary situations. (source: Personal Data Protection Act 2012 - Singapore Statutes Online.md)

The captured table of contents identifies breach notification (Part 6A), offences affecting personal data/anonymised information (Part 9B), and enforcement/appeals parts. (source: Personal Data Protection Act 2012 - Singapore Statutes Online.md)
