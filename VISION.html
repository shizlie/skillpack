Vision · mid-term · 2026 → 2027 · composable

Skills are software. Pay the people who build them.

skillpack is not one product. It is a set of small, separable components — signer, leaser, runtime, ledger, billing, hub, gateway, CLI — that snap together into the operating model a skill author needs to ship into regulated, disconnected, customer-controlled environments. The hub is one delivery channel. A skill can also ship by file, by email, by USB, by internal mirror. Every component does one job and exposes a contract.

01 · the unbundling

One platform, many components. Pick what you need.

The natural failure mode for a control plane is to merge "everything the vendor needs" into a single service: signing, leasing, distribution, billing, hosting, the dashboard, the registry. That works until a buyer requires self-host. Or rips out the registry. Or wants their own payment provider. Or air-gaps the entire environment and only accepts skills delivered on a USB drive.

skillpack inverts the default. Every responsibility is its own component with its own package, its own contract, its own deploy. You can run all of it. Or one piece. Or replace any of it with your own.

The hub helps. It is not required. The leaser is required. The signer is required. Everything else is a substitution.

03 · delivery is plural

The hub is one channel. Skills also ship by file, mail, USB, internal mirror.

A signed .mcpb is portable by design. Sign once. Ship by whatever the customer's environment allows. The leaser, the runtime, and the ledger don't care which channel moved the bytes — they care that the signature verifies, the lease matches, and the meter syncs.

Channel How the bytes move Trust anchor Update path
hubdiscovery + auto-pull Agent or runtime fetches by entitlement from a public or self-hosted registry. Bundle signature + hub manifest signature. Channel promote (stable / beta) + delta updates.
direct downloadvendor URL Vendor publishes a signed .mcpb on their own site. Buyer downloads. Bundle signature alone. Operator re-downloads on tag bump.
emailattachment Vendor mails the bundle + public key + SHA256 sums to the customer's named operator. Bundle signature + out-of-band key fingerprint. New email per release.
file dropSFTP, S3, share Vendor places signed artifacts in a customer-owned drop. Customer pulls on schedule. Bundle signature + drop-side access control. Pull job watches for new tag.
removable mediaUSB, optical Air-gapped sites: physically carried sealed media. Mandatory for some defense and lab environments. Bundle signature + chain-of-custody log. Scheduled sneakernet rotation.
internal mirrorcustomer-hosted hub Customer runs a self-hosted hub mirror. Authors publish there; agents fetch locally only. Bundle signature + mirror trust policy. Mirror sync job (or manual).
OCI / package registryexisting infra Treat the .mcpb as an artifact in OCI, Artifactory, or similar. Bundle signature + registry policy. Existing CI/CD pull paths.

04 · composition diagram

Components plug into a shared contract bus.

The middle of the system is not a service — it is a set of contracts: signed bundle, lease + policy, meter event, ledger row, pricing rule. Anything that speaks a contract can replace anything else that speaks the same contract.

05 · example compositions

Same components. Four different deployments.

No fork. No new code. Just choose which components you run, and which channel moves the bytes.

A · hosted full stack

Vendor sells SaaS-style, customers connect

All provider-side components run on skillpack hosted. Hub publishes. Runtime pulls by entitlement. Direct upload to ledger.

signer→ leaser→ hub→ runtime→ ledger→ billing
B · self-hosted vendor

Vendor runs the leaser, no hub

Vendor self-hosts leaser + ledger + billing in their own infra. Skills ship by direct download from the vendor site. No hub anywhere.

signer→ leaser (self)→ (no hub)→ runtime→ ledger (self)
C · air-gapped customer

USB-delivered, gateway-synced

Skill ships via sealed USB. Customer site runs an edge gateway. Meter uploads go to the gateway, then batched out on a scheduled sync window. Leaser is reachable through one approved egress.

signer→ USB→ runtime→ edge gateway→ leaser→ ledger
D · internal-only enterprise

Customer-hosted hub mirror

Customer runs a self-hosted hub mirror that talks to the vendor leaser. Internal authors publish to the mirror; nothing leaks out. Per-tool entitlements gated by the customer's own SSO.

signer→ hub mirror (self)→ runtime→ leaser→ ledger

06 · the three people who must win

Creator. Leasee. User. None of them win today.

A skill only becomes a real economy when all three are rewarded by the same loop. Below: who they are, what they want, what they get today, and what changes when the loop closes.

M

Maya · Skill Creator

Solo specialist · 12 yrs commercial-contracts attorney · ships one skill

Wants

  • Royalties when her skill runs anywhere.
  • Attribution that survives forks.
  • Channel to push a fix the same day she writes it.
  • IP protection — her prompts, her clause library.
  • Proof of quality buyers can trust.

Today

  • Publishes a .md file on GitHub.
  • Zero royalty. Zero usage signal.
  • Forked silently, embedded into vendor agents.
  • No update path. Can't fix the v0.3 bug at the firm using her v0.1.

With skillpack

  • Signs the bundle. Provenance is hers.
  • Ledger shows every accepted tool call by leasee, workspace, tool.
  • Pricing rule + payout per call (later: per outcome).
  • Push a patched .mcpb; entitled seats fetch on next launch.
  • Signed eval suite travels with the bundle.
K

Kuro · Skill Leasee

IT + procurement lead · mid-size law firm · 240 lawyers · regulated data

Wants

  • Vetted skills with provable provenance.
  • Per-seat, per-tool entitlements he controls.
  • Audit log for compliance and incident review.
  • Predictable cost; no surprise overages.
  • Air-gap-compatible. Skills run inside the firm.

Today

  • Markdown skills pasted into chat. No idea who wrote them.
  • No way to revoke without breaking a lawyer's day.
  • No usage analytics, no procurement story.
  • Cloud-only vendors blocked by client confidentiality.

With skillpack

  • Leases by workspace + seat. Entitlements per tool.
  • Skills delivered by hub, internal mirror, or USB — his choice.
  • Local meter chain → his audit log.
  • Lease + 72h grace = humane kill-switch, no broken Mondays.
  • Self-hosted leaser keeps every byte inside the firm.
P

Priya · Skill User

Senior associate · uses an agent daily · doesn't want to think about licensing

Wants

  • The skill works. Today. In her agent.
  • Latest version, no manual update step.
  • Trust the answer — citations, version, author.
  • No "vendor is down" excuses on a Friday filing.

Today

  • Pastes prompts she copied from a colleague's chat.
  • Different lawyers running different versions silently.
  • No idea if the clause comes from a vetted source.
  • Cloud outage means her skill stops mid-draft.

With skillpack

  • Agent fetches her entitled skills on launch. She sees nothing.
  • Runs locally. Cloud outage = grace window, work continues.
  • Every answer carries skill ID + version in the agent UI.
  • If Maya pushes a fix, Priya gets it next session.

08 · what the creator actually gets

Five rewards. One flywheel. Each spin compounds.

Money is the obvious one and not the only one. The platform rewards the creator on five axes; each strengthens the next. None of it requires the creator to be a company, run a server, or chase invoices.

Royalty

Every accepted tool call in the ledger turns into a payout against a pricing rule. Per-call today; per-outcome later. Royalty splits across co-authors land in H5.

Reputation

Signed eval suites attached to the bundle generate badges. The hub ranks skills by independently-attested quality, not stars. Maya's v1.4 climbs because her evals pass on real buyer corpora.

Signal

For the first time, the creator sees which tool is used, by what kind of workspace, how often. That signal points her to v1.5 — and tells her which 30% of the skill nobody touches.

Control

Signing proves authorship. Lease + grace means she can stop renewing without breaking a lawyer's day. Bundle archives knowledge sealed — no plain markdown leaking.

Reach

Hub publishes for discovery. Internal mirrors carry the skill into firms that would never have found her. File, email, USB all carry the same signed artifact. The skill travels; the contract follows.

The flywheel. Reach → more meter events → more signal → better v-next → higher eval score → more reputation → bigger channel placement → more reach. Royalty rides every step. The creator never has to operate a server, chase an invoice, or beg a buyer for usage numbers.

09 · north star

One sentence to test every decision against.

A skill is software. The person who built it should be paid every time it runs — in regulated, offline, sovereign environments where "phone home" is not an option, on a platform small enough that any operator can run any component themselves, and composable enough that the hub is one option among many.

If a feature can't be drawn as a single component with a sharp contract, or it breaks self-host parity, or it forces the hub and the leaser into the same process — it doesn't ship.
